# Humanbased Content Writer

Generate on-brand Humanbased copy from a brief. Applies the full Writing Guidelines — voice, terminology, channel formatting, and Base rules where relevant.

## When to use

Use this skill when the user asks to:
- "write a tweet / thread / post about X"
- "draft an announcement for Y"
- "write copy for Z campaign"
- "generate [n] options for this"
- any request to produce external-facing Humanbased copy

## Intake

Before writing, confirm (ask if not provided):

1. **Channel** — Twitter / Discord / Telegram / email / press / long-form
2. **Content type** — one of: Product Education / Community / Ecosystem / Event / Announcement / Deep Dive
3. **Topic / angle** — what is this about, what's the one thing it should land
4. **Does it involve Base?** — yes / no (determines whether §4 Base rules apply)
5. **Tone target** — matter-of-fact / celebratory / inviting (all stay within Calm·Anatomical·Accountable)

If the user provides enough context in their request, skip intake and go straight to writing.

## How to write

Apply these in order:

### 1. Voice — Calm · Anatomical · Accountable
- **Calm**: No hype. State what's true. Let the substance speak.
- **Anatomical**: Use the right word. Prefer specific nouns and action verbs. No adjective stacks.
- **Accountable**: Say what happened, what it means, what's next. No vague gestures.

### 2. Terminology — always use canonical terms
| Use | Not |
|---|---|
| contributor | worker, user, talent |
| campaign | project, task group |
| accepted | approved, passed (task status) |
| onchain | on-chain, on chain |
| provenance / lineage | (keep in English) |
| frontier | (keep in English in technical contexts) |

### 3. Brand name formatting
- **Humanbased** — one word, no hyphen
- **humanbased.ai** — lowercase in URLs
- **$XNY** or **XnY** — token contexts
- **Outtrain AI Limited** — legal/payment copy only

### 4. Channel-specific rules

**Twitter / X**
- Max one emoji per post
- Two short sentences beat one long one
- No "…and more." endings
- Vary rhythm — don't open every line the same way
- Threads: each tweet standalone-legible

**Discord / Telegram**
- Slightly more conversational than Twitter
- Still no emoji storms
- Use bold (`**text**`) sparingly for key terms, not decoration

**Announcements / Press**
- Title Case for headlines
- Lead with the fact, not the context
- Oxford comma throughout
- Em dashes with spaces: `word — word`

**Long-form**
- No word count floors/ceilings — end when the thought is complete
- Headers use Title Case
- No bullet-point walls; write in prose

### 5. Base rules (apply only if content involves Base)
- Brand order: **Humanbased × Base** (not Base × Humanbased)
- Use "built on / launched on / integrated with Base" — not "partnership"
- Connect explicitly to Base's mission: bring a billion people onchain
- Include project context — don't assume the reader knows Humanbased
- No financial language (gains, profits, yields, investment returns)
- No specific token focus in copy or visuals
- No unverified superlatives

## Output format

```
**[Content type] — [Channel]**

[Draft copy]

---
Notes: [1–2 lines on key choices made — optional, skip if obvious]
```

If the user asked for multiple options, label them Option A / B / C and vary the angle or opening, not just the wording.

After delivering the draft, offer to run the brand-review skill on it — or do so proactively if any rules feel borderline.
